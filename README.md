# Electricity Demand Forecast — Group 12

Forecasts electricity demand for 181 countries over the next 20 years.
Built with Linear Regression, served through a REST API, and deployed via Docker.

---

## What You Need Before Starting

- Python 3.10 or higher
- pip
- Docker (only for the deployment section)

---

## Step 1 — Clone the Repo

```bash
git clone https://github.com/Mitch-json/AISD-Project.git
```

---

## Step 2 — Install Dependencies

There are two `requirements.txt` files in this project:

| File | Purpose |
|------|---------|
| `requirements.txt` | For running the notebooks (data processing, model training) |
| `group12-model-api/requirements.txt` | For running the API only |

Install the notebook dependencies first:

```bash
pip install -r requirements.txt
```

---

## Step 3 — Run the Notebooks

Open Jupyter:

```bash
jupyter notebook
```

Run the notebooks **in this exact order**. Each one produces files the next one needs.

---

### Notebook 1: `data_preprocessing.ipynb`

Cleans the raw energy dataset.

- **Reads:** `owid-energy-data.csv` (already in the repo)
- **Outputs:** `df_clean.csv`

Run all cells top to bottom.

---

### Notebook 2: `model_selection.ipynb`

Compares three models — Linear Regression, SARIMAX, and LSTM.

- **Reads:** `df_clean.csv`
- **Shows:** Accuracy metrics (RMSE, MAE) for each model

Linear Regression was selected for production because it gives the best balance of accuracy, speed, and interpretability.

---

### Notebook 3: `final_model.ipynb`

Trains the final model and generates forecasts.

- **Reads:** `df_clean.csv`
- **Outputs:**
  - `group12-model-api/model.pkl` — trained Linear Regression model
  - `group12-model-api/scaler.pkl` — feature scaler
  - `electricity_forecast_clean.json` — forecasts for all 181 countries

Run all cells top to bottom.

---

## Step 4 — Set Up the API

After the notebooks finish, copy the cleaned data into the API folder:

```bash
cp df_clean.csv group12-model-api/
```

Install the API dependencies:

```bash
pip install -r group12-model-api/requirements.txt
```

---

## Step 5 — Run the API Locally

```bash
cd group12-model-api
uvicorn main:app --host 0.0.0.0 --port 8000
```

The API takes a few seconds to start — it computes and caches forecasts for all 181 countries on launch.

Once running:
- **Interactive docs:** `http://localhost:8000/docs`
- **Test it:** `curl http://localhost:8000/health`

---

## API Endpoints

| Endpoint | What it returns |
|----------|-----------------|
| `GET /health` | Confirms the API is running |
| `GET /countries` | List of all 181 country names |
| `GET /forecast` | Forecasts for all 181 countries |
| `GET /forecast/{country}` | Forecast for one specific country |

**Example:**

```bash
curl http://localhost:8000/forecast/Kenya
```

```json
[
  { "year": 2023, "demand": 9.1, "type": "historical", "gdp": 118000000000, "population": 55000000 },
  { "year": 2024, "demand": 9.4, "type": "forecast", "gdp": 122000000000, "population": 56000000 }
]
```

Each record includes `year`, `demand` (TWh), `type` (historical or forecast), GDP, population, and 12 electricity generation breakdown fields.

> Country names are case-sensitive. Use `/countries` to see exact spellings.

## Step 6 — Deploy with Docker

Make sure Docker is installed and running, then from the API folder:

```bash
cd group12-model-api

docker build -t group12-model-api .

docker run -p 8000:8000 group12-model-api
```

API is now available at `http://localhost:8000`.

## Step 7 — Deploy to the Cloud

### Option A — Render (simplest)

1. Push this repo to your own GitHub account
2. Go to [render.com](https://render.com) and create a **New Web Service**
3. Connect your GitHub repo and set:
   - **Root Directory:** `group12-model-api`
   - **Environment:** Docker
   - **Port:** `8000`

Render will build and host the container automatically.

## Responsible AI

### What the model does

The model takes a country's historical electricity demand, GDP, and population and extrapolates a 20-year trend. It uses Linear Regression — meaning its predictions are fully explainable. You can see exactly how much each input (GDP, population, past demand) contributes to each forecast.

### What the model does not do

The model extrapolates trends. It cannot anticipate:
- Policy changes (carbon taxes, electrification targets)
- Technology shifts (mass EV adoption, new grid capacity)
- Economic shocks (recessions, collapse like Lebanon 2019+)

Forecasts beyond 5–10 years should be treated as trend projections, not precise predictions.

### Data quality varies by country

Countries with complete long-term records produce more reliable forecasts. For countries with missing GDP or demand data, values were estimated using regional medians or trend extrapolation. Forecasts for these countries carry more uncertainty.

### The model does not self-update

It was trained on data through 2024. It does not automatically incorporate new data. Retrain it using the notebooks when updated data is available.

### Use these forecasts responsibly

- Do not use for real-time grid operations or safety-critical decisions
- Share the model's test accuracy when presenting results — test RMSE is ~58.75 TWh, MAE is ~23.01 TWh
- For long-range forecasts (beyond 2030), present them as scenarios rather than predictions
- Note when a country's data was imputed rather than directly measured

## Troubleshooting

**API returns 404 for a country** — Names are case-sensitive. Check `/countries` for the exact spelling.

**`model.pkl` or `scaler.pkl` not found** — Run `final_model.ipynb` to generate them.

**`df_clean.csv` not found in API folder** — Run `cp df_clean.csv group12-model-api/` from the project root.

**API is slow to start** — This is expected. It pre-computes all 181 country forecasts on startup. Requests after that are instant.

**Docker build fails** — Confirm `model.pkl`, `scaler.pkl`, and `df_clean.csv` are all inside `group12-model-api/` before building.
